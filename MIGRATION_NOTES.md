## Environment Variables Required

### Root Level (`.env` or `.env.local`)
Currently unused - environment is managed per-package.

### Server Package (`packages/server/.env`)
Required for backend operation:
```
FTC_API_KEY=                           # Your FTC Events API key (Base64 encoded)
DATABASE_URL=postgres://user:pass@localhost:5432/dbname
PORT=4000
LOGGING=0                              # Enable TypeORM SQL logging (1=true)
SYNC_DB=1                              # Auto-sync database schema (1=true)
SYNC_API=1                             # Auto-sync with FTC API (1=true)
CACHE_REQ=1                            # Cache API requests (1=true)
FRONTEND_CODE="anystring"              # Header identifier for frontend requests
DB_TIMEOUT=5000                        # Database request timeout (ms)
```

### Web Package (`packages/web/.env`)
Required for frontend operation:
```
PUBLIC_SERVER_ORIGIN="localhost:4000"  # Backend URL
PUBLIC_FRONTEND_CODE="anystring"       # Must match server's FRONTEND_CODE
```

## Running FTCScout Locally

### Prerequisites
1. PostgreSQL database setup (see `instructions/instructions.md` for setup commands)
2. FTC Events API credentials (see `instructions/instructions.md` for setup)
3. Node.js 20+ (Canvas dependency may have issues with later versions)
4. npm with workspace support (npm 7+)

### Development Workflow

**Terminal 1: Build common package (watch mode)**
```bash
npm run common:watch
```

**Terminal 2: Build server (watch mode)**
```bash
npm run server:watch
```

**Terminal 3: Run server (with nodemon)**
```bash
npm run server:dev
# Waits for dist files, then watches for changes
# Server will start downloading FTC API data on first run (takes a while!)
```

**Terminal 4: Generate GraphQL types & start frontend (after server is ready)**
```bash
npm run web:gen     # Generate GraphQL types from server schema
npm run web:dev     # Start Vite dev server
# Frontend will be available at http://localhost:5173 (or similar)
```

### Production Build
```bash
npm run common:build
npm run server:build
npm run web:build   # Creates production SSR bundle in packages/web/.svelte-kit/
```

### TypeScript Checking
```bash
npm run check       # Check all packages
npm run web:check   # Check web package only
npm run web:check:watch  # Watch mode
```

## Known Issues & Limitations

### TypeScript Check Errors (Expected)
The `npm run web:check` command shows ~290 errors related to GraphQL types:
- **Root Cause**: Stub `graphql-operations.ts` file has incomplete type definitions
- **Resolution**: Run `npm run server:dev` and `npm run web:gen` to generate real types from live server
- **Impact**: TypeScript checking will be stricter after codegen runs
- **Build Status**: ✅ **NOT affected** - `npm run web:build` succeeds because Vite doesn't enforce strict type checking

### GraphQL Codegen Error
Running `npm run web:gen` without a running server will fail:
```
Failed to load schema from http://localhost:4000/graphql: Connection refused
```
- **Solution**: Start the server first with `npm run server:dev` before running codegen

### Database & API Setup
The app will fail to start without:
- ✅ PostgreSQL running and configured
- ✅ FTC Events API credentials (the app needs real data)
- ✅ Environment variables properly set

See `instructions/instructions.md` for detailed setup.
